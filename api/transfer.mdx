Base URL: `https://api.onbeam.io`

Transfers move tokens between addresses and chains. The API can source liquidity from any wallet in the project using aggregated balances.

## Create transfer

Method: `POST`  
Path: `/transfers`

Creates a cross-chain or same-chain transfer funded by a wallet.

### Request body

- `fromWalletId` (string, required): Wallet identifier or address funding the transfer.
- `toAddress` (string, required): Recipient address on the destination chain.
- `fromChainId` (integer, required): Source chain ID.
- `toChainId` (integer, required): Destination chain ID.
- `token` (string, required): Token address or symbol to transfer.
- `amount` (string, required): Token amount as a decimal string.
- `priority` (string, optional): `standard` or `fast`, impacting routing and fees.
- `metadata` (object, optional): Free-form metadata attached to the transfer.

### Response

- `transferId` (string): Unique transfer identifier.
- `status` (string): Initial status, for example `pending`.
- `from` (object): Source chain, token, and amount.
- `to` (object): Destination chain, token, and amount.

## Get transfer

Method: `GET`  
Path: `/transfers/{transferId}`

Fetch details and status for a specific transfer.

### Path parameters

- `transferId` (string, required): Transfer identifier returned from `/transfers`.

### Response

- `transferId` (string)
- `status` (string): One of `pending`, `in_progress`, `completed`, or `failed`.
- `transactions` (array): On-chain transactions associated with the transfer.
- `error` (object, nullable): Error details if the transfer failed.
