Base URL: `https://api.onbeam.io`

Swap and transfer combines a swap and a transfer into a single atomic operation. This is useful when users fund in one asset and chain but want to receive a different asset on another chain.

## Create swap-and-transfer

Method: `POST`  
Path: `/swap-and-transfer`

Executes a swap from a source asset to a destination asset and delivers the proceeds to a recipient.

### Request body

- `fromWalletId` (string, required): Wallet identifier or address funding the operation.
- `fromChainId` (integer, required): Source chain ID.
- `toChainId` (integer, required): Destination chain ID.
- `fromToken` (string, required): Source token address or symbol.
- `toToken` (string, required): Destination token address or symbol.
- `amount` (string, required): Amount of source token to swap.
- `recipient` (string, required): Final recipient address on the destination chain.
- `slippageBps` (integer, optional): Maximum allowed slippage in basis points.
- `priority` (string, optional): `standard` or `fast`.
- `metadata` (object, optional): Free-form metadata for the operation.

### Response

- `operationId` (string): Unique identifier for the combined operation.
- `status` (string): Initial status of the operation.
- `from` (object): Source chain, token, and amount.
- `to` (object): Destination chain, token, and estimated amount.

## Get swap-and-transfer status

Method: `GET`  
Path: `/swap-and-transfer/{operationId}`

Fetch the latest status of a combined swap-and-transfer operation.

### Path parameters

- `operationId` (string, required): Identifier returned from `/swap-and-transfer`.

### Response

- `operationId` (string)
- `status` (string): One of `pending`, `in_progress`, `completed`, or `failed`.
- `transactions` (array): On-chain transactions that make up the operation.
- `error` (object, nullable): Error details if the operation failed.
